  ## 目标

  编写 bash + curl 冒烟脚本 `backend/scripts/smoke_nodetree.sh`，
  覆盖 M4 Epic 的全部后端 API 路径：节点 CRUD、权限设置、
  权限过滤（none 用户看不到节点）、移动、软删除、恢复、
  权限组 CRUD、成员管理、改角色。
  
  脚本在 `docker compose up` 启动三服务后直接运行，
  全部用例通过即可关闭 M4 里程碑。

  ## 参考

  严格参考已有的 `backend/scripts/smoke_auth.sh` 的风格和工具函数：
  - `split_response()` — 从 curl 输出分离 body 和 status code
  - `extract_json()` — 从 JSON body 提取字段（jq 优先，fallback node -e）
  - `assert_status()` — 断言 HTTP status code，统计 PASS/FAIL
  - `request()` — 封装 curl 调用（method, path, body, token）

  #### 测试步骤详细实现

  **准备阶段**

  ```
  STEP: Admin 登录
    request POST "/api/auth/login" '{"email":"$ADMIN_EMAIL","password":"$ADMIN_PASSWORD"}'
    assert_status "admin login" 200
    ADMIN_TOKEN=$(extract_json "$RESPONSE_BODY" '.data.access_token')

  STEP: 注册 Member
    request POST "/api/auth/register" '{"email":"$MEMBER_EMAIL","password":"$MEMBER_PASSWORD","display_name":"$MEMBER_NAME"}'
    assert_status "register member" 201
    MEMBER_TOKEN=$(extract_json "$RESPONSE_BODY" '.data.access_token')

  STEP: 获取 Member 信息（拿 user_id 用于后续权限设置）
    request GET "/api/me" "" "$MEMBER_TOKEN"
    assert_status "get member me" 200
    MEMBER_ID=$(extract_json "$RESPONSE_BODY" '.data.id')
  ```

  **T01: Admin 创建根文件夹**

  ```
  request POST "/api/nodes" '{"parent_id":null,"kind":"folder","title":"测试文件夹"}' "$ADMIN_TOKEN"
  assert_status "T01 create root folder" 201
  FOLDER_ID=$(extract_json "$RESPONSE_BODY" '.data.id')
  ```

  **T02: Admin 在文件夹下创建子文档**

  ```
  request POST "/api/nodes" '{"parent_id":"'"$FOLDER_ID"'","kind":"doc","title":"测试文档"}' "$ADMIN_TOKEN"
  assert_status "T02 create child doc" 201
  DOC_ID=$(extract_json "$RESPONSE_BODY" '.data.id')
  ```

  **T03: Admin 列出根节点子节点**

  ```
  request GET "/api/nodes" "" "$ADMIN_TOKEN"
  assert_status "T03 list root children" 200
  # 验证返回的 data 是数组且包含 FOLDER_ID
  ```

  **T04: Admin 获取单个节点**

  ```

  ```

  **T05: Admin 重命名节点**

  ```

  ```

  **T06: Admin 创建第二个根文件夹并移动文档到新文件夹**


  ## 编码规范

  - bash strict mode: `set -euo pipefail`
  - 所有变量用双引号包裹
  - JSON body 中嵌入变量时用 `'"$VAR"'` 的 quote 拼接模式（参考 smoke_auth.sh 现有风格）
  - 脚本文件头: `#!/usr/bin/env bash`
  - 每个 request() 调用后紧跟 split_response
  - assert_status 的 label 用可读的英文描述
  - 不要在脚本中 hardcode UUID，全部从前序请求的 response 中提取
  - 文件末尾保留一个空行
